How operators save lines

Operators such as the coälescing (or null-coälescing) operator in Microsoft’s C# language are capable of saving you from unnecessary lines of code when writing if-then-else assignment code blocks. In C# the operator is ?? and it’s also called the “double question mark” operator, but, unlike the ternary “single question mark operator” ?:, ?? only takes two operands; a value to be assigned provided it isn’t null, and an alternative for when it is.

An example in C#:

using System;

class Cafeteria {
    static void Main(string[] args) {
        serve(null);
    }

    static void serve(string requested) {
        string toServe = requested ?? "French Fries";
        Console.WriteLine(toServe);
    }
}

Centering a div, the proper way

Consider <div class="container">Center this block, but not this text.</div>. Everybody is centering their div’s wrong:

  1. Setting margin-left and margin-right will work, but it won’t be IE6 compatible.

  2. Setting the div’s parentNode class to have text-align: center and then setting the div to have text-align: left (or right, or justify) is also incorrect. You’d have to add content { width: 640px; } for that, but that stops the div from responding to changes in the browser width. You cannot specify max-width for the .content because that also won’t be IE6 compatible.

  3. Using JavaScript disables the better browsers from having @media queries apply to these .content div’s.

  4. And, using IE expressions is just wrong.

Benchmarking algorithms

When a new algorithm is proposed, it is commonly demonstrated to outperform others in a benchmark using a set of randomly generated or commonly used test instances from the problem’s literature. The No Free Lunch (NFL) theorem1 tells us it’s impossible for one single algorithm to outperform all others on all instances of a problem2, so claims of better performance using this common methodology only hold for the instances tested and those the tested instances are representative of.

The reason proposed algorithms cannot proclaim superior performance across all instances is because any increase in performance is only obtained by exploiting particular characteristics of particular problem instances.2,3,4 An example that will be familiar with many is how a sorting algorithm performs. The time complexity for Selection Sort is $\mathcal{O}(n^2)$ for the best case, the worst case and the average case. In a best case scenario, Insertion Sort will demonstrate improved performance and complete its comparisons in $\mathcal{O}(n)$ linear time. Merge sort and Quicksort more so over, do better on average and complete in $\mathcal{O}(n\log n)$ time, with Mergesort maintaining this linearithmic complexity in the worst case. Quicksort can have quadratic time complexity in, again, certain instances of the (sorting) problem.

  1. David H. Wolpert, and William G. Macready, No Free Lunch Theorems for Optimization. IEEE Transactions on Evolutionary Computation, Vol. 1, Iss. 1, pp. 67-82 (April 1997). ↩

  2. Ibid., pp 67, “any elevated performance over one class of problems is offset by performance over another class.” ↩ ↩2

  3. Yu-Chi Ho and David L. Pepyne, Simple Explanation of the No-Free-Lunch Theorem of Optimization. IEEE Conference of Decision and Control (Proc. of 40th), Vol. 5, pp. 4409-4414 (December 2001). Orlando FL, USA. ↩

  4. Ibid., p. 4409, “a general-purpose universal optimization strategy is impossible, and the only way one strategy can outperform another is if it is specialized to the structure of the specific problem under consideration.” ↩

The Algorithm Selection Problem

The Algorithm Selection Problem (ASP) involves selecting an algorithm, from among those in a portfolio, that will perform effectively or good or best on a problem instance.

It is important to make the distinction between a problem and its instances. A problem is the requirement to find an output subject to constraints. An instance of a problem is one of many (possibly infinite) configurations of the problem and acts as the input to to which the constraints are applied.

Symbols and definitions used

The TSP model1 consists of the input graph which can be either:

  • a complete undirected graph $G=(V,E)$
  • or a complete directed graph $G=(V,A)$

where

  • $V=\{1,\ldots, n\}$ is the set of vertices or locations or ‘cities’,
  • $S$ is a non-empty proper subset of $V$,
  • $E=\{(i,j):i,j\in V,i<j\}$ is the set of edges,
  • $A=\{(i,j):i,j\in V,i\ne j\}$ is the set of arcs,
  • $C=(c_{i,j})$ is the cost matrix defined on $E$ or $A$,
