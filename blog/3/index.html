The Traveling Salesman Problem


Symbols and definitions used

The TSP model1 consists of the input graph which can be either:

  • a complete undirected graph $G=(V,E)$
  • or a complete directed graph $G=(V,A)$

where

  • $V=\{1,\ldots, n\}$ is the set of vertices or locations or ‘cities’,
  • $S$ is a non-empty proper subset of $V$,
  • $E=\{(i,j):i,j\in V,i<j\}$ is the set of edges,
  • $A=\{(i,j):i,j\in V,i\ne j\}$ is the set of arcs,
  • $C=(c_{i,j})$ is the cost matrix defined on $E$ or $A$,
  • $k$ is the total cost of the tour,

Motivation

The Traveling Salesman Problem (TSP), of finding the shortest tour through $n$ cities, visiting each exactly once and finishing at the starting city, is classified as an $\mathcal{NP}$-hard problem whose solutions may require unfeasible computational resources or running times even for real-world instances. By its definition, a solution to a TSP instance is not just finding a ‘good’ tour, or one better than any previously known, but is the finding of the absolute shortest tour and determining that there is no better tour possible1. Applications of the TSP in fields such as logistics, telecommunications routing and microchip manufacturing often lack the need to strictly satisfy these requirements and finding just ‘short’ feasible tours is often the case. The result has been the interest in, and development of large numbers of TSP heuristic methods that yield near-optimal results.

  1. Wiliam Cook, TSP Lower Bounds: Certifying Tour Quality (May 2005). Georgia Institute of Technology, Atlanta GA, USA. ↩

Abstract

The Traveling Salesman Problem (TSP) is an $\mathcal{NP}$-hard problem easy to describe but difficult to solve. Given $n$ cities, a salesman must complete a tour by visiting each city exactly once and then return to their starting city. Finding the shortest tour length is a combinatorial optimization problem that traditional computational methods have difficulty with typically for large $n$.1 The cause of problem difficulty is not a single problem-specific characteristic, but rather several characteristics of the problem instances.2,3,4

Basic sorting algorithms

Bubble sort, selection sort and insertion sort all written in C for ascending order1. Kept as minimal as possible so it’s as easy as possible for adaptation to your needs.

Bubble sort

Loop through the list, swapping any adjacent pair of elements if the first is larger. Repeat looping $n - 1$ times (end swapping one place earlier each time).

void bubble(int a[], int n) {
    int i, j, temp;

    for (i = 0; i < n - 1; i++)
        for (j = 0; j < n - 1; j++)
            if (a[j] > a[j + 1])
                temp = a[j], a[j] = a[j + 1], a[j + 1] = temp;
}

Using SSH keys

Using SSH keys

  1. Generate a key pair with ssh-keygen. Place:
    • public key on server: ~/.ssh/authorized_keys.
    • private key on client: ~/.ssh/id_rsa.
  2. Make your private key private (mode 600 = only you the owner can read/write): chmod 600 ~/.ssh/id_rsa.

If you generate a key pair without a passphrase, anybody who captures your private key can access servers with its public key counterpart.
