The Algorithm Selection Problem

The Algorithm Selection Problem (ASP) involves selecting an algorithm, from among those in a portfolio, that will perform effectively or good or best on a problem instance.

It is important to make the distinction between a problem and its instances. A problem is the requirement to find an output subject to constraints. An instance of a problem is one of many (possibly infinite) configurations of the problem and acts as the input to to which the constraints are applied.

Rice’s framework1 introduces an abstract model to clarify consideration of the ASP. The abstract model2 is shown in Figure 1 and its items are described below.

problem space $\mathcal{P}$
This is the problem at hand. $\mathcal{P}$ may represent the traveling salesman problem, the knapsack problem, graph colouring, or any other given problem.
problem instance $x$
A particular configuration of $\mathcal{P}$ to be solved.
feature space $\mathcal{F}$
Contains measurable instance characteristics.
instance features $f(x)$
The measured features of instance $x$ which map it from $\mathcal{P}$ to $\mathcal{F}$.
algorithm selection $S(f(x))$
The selection mapping features to $\alpha$.
$\alpha$
An algorithm that solve instances of problem type $\mathcal{P}$.
algorithm space $\mathcal{A}$
The portfolio of candidate algorithms. When counting algorithms, those which are otherwise identical except in some numeric parameter are not considered different.
performance space $\mathcal{Y}$
The space in which algorithm performance is measured.
performance mapping $y$
How algorithm $\alpha$ performed on instance $x$.

Within Rice’s framework, our goal to solve the ASP can be stated as:

Given problem instance $x\in\mathcal{P}$ with features $f(x)\in\mathcal{F}$, find the algorithm selection mapping such that the selected algorithm $\alpha\in\mathcal{A}$ maximizes the performance mapping $y(\alpha, x)\in\mathcal{Y}$.

Note that the selection mapping is $S(f)$ and depends only on features, yet it is the problem instance $x$ that has its performance evaluated.

  1. John R. Rice, The Algorithm Selection Problem, Computer Science Technical Reports, Paper 99 (1975). Purdue University, West Fayette IN, USA. ↩

  2. Ibid, p. 9. This is Rice’s model with selection based on features – an improvement of a basic model lacking features but otherwise identical. Rice introduced a basic model (p. 3) and supplemented it with features only to systematize the introduction of problem subclasses. ↩
