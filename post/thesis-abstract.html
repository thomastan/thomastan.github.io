Abstract

The Traveling Salesman Problem (TSP) is an $\mathcal{NP}$-hard problem easy to describe but difficult to solve. Given $n$ cities, a salesman must complete a tour by visiting each city exactly once and then return to their starting city. Finding the shortest tour length is a combinatorial optimization problem that traditional computational methods have difficulty with typically for large $n$.1 The cause of problem difficulty is not a single problem-specific characteristic, but rather several characteristics of the problem instances.2,3,4 Many statistical properties of TSP instances do indeed influence difficulty, and evolutionary computing can be used to identify such influences.5 I present a new technique for visualizing algorithms and show how this develops a greater insight of algorithm performance in a TSP case study. The results show that even for other combinatorial optimization problems, this deeper understanding of algorithms equates to better analysis and performance prediction. The algorithm selection problem is solved with an automated choice for the algorithm being the one predicted to perform best.
