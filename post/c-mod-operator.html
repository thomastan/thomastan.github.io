C's mod operator yields negative integers

What is -10 % 6? In C99 that’s -4. For the mathematical Euclidean definition:

for $a = -10$ and $q = 6$, $r = 2$ ($n = -1$).

It turns out C99 follows Donald Knuth’s definition with the floor function where the remainder $r$ is always positive. In C89, the result is undefined for $a \oplus n < 0$ and is implementation specific. The C89 result is usually that the integer/integer division truncates, as most CPUs do this, and so C99 standardized the de facto, assigning the result the same sign as the dividend.
